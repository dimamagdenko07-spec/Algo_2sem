# lab10
Лабораторная работа №10 - Динамическое программирование. Экономим расточительствуя

Динамическое программирование — это когда вы решаете задачу один раз, записываете ответ, и больше никогда не решаете её снова. Звучит как мечта любого студента на экзамене, но увы, работает только для алгоритмов.

Название "динамическое программирование" придумал Ричард Беллман в 1950-х. Почему "динамическое"? Потому что слово звучало круто и помогало получить финансирование от военных. Серьёзно, он сам это признавал. Так что если кто-то спросит, почему оно так называется — теперь вы знаете: маркетинг.

А "экономим расточительствуя" — это про главный принцип ДП: мы тратим память, чтобы сэкономить время. Запоминаем решения подзадач вместо того, чтобы вычислять их снова и снова.

### Теоретическая справка

Динамическое программирование применимо, когда задача обладает двумя свойствами:

1. **Оптимальная подструктура** — оптимальное решение задачи можно построить из оптимальных решений подзадач
2. **Перекрывающиеся подзадачи** — одни и те же подзадачи решаются многократно

**Два подхода к реализации:**

| Подход | Описание | Плюсы | Минусы |
|--------|----------|-------|--------|
| **Мемоизация** (сверху вниз) | Рекурсия + кэширование результатов | Вычисляем только нужные подзадачи | Накладные расходы на рекурсию |
| **Табуляция** (снизу вверх) | Итеративное заполнение таблицы | Нет рекурсии, предсказуемая память | Вычисляем все подзадачи |

**Классический пример — числа Фибоначчи:**

```c
// Наивная рекурсия: O(2^n) — экспоненциальный ужас
int fib_naive(int n) {
    if (n <= 1) return n;
    return fib_naive(n - 1) + fib_naive(n - 2);
}

// Мемоизация: O(n) времени, O(n) памяти
int fib_memo(int n, int* cache) {
    if (n <= 1) return n;
    if (cache[n] != -1) return cache[n];
    cache[n] = fib_memo(n - 1, cache) + fib_memo(n - 2, cache);
    return cache[n];
}

// Табуляция: O(n) времени, O(n) памяти
int fib_tab(int n) {
    int* dp = malloc((n + 1) * sizeof(int));
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];
    int result = dp[n];
    free(dp);
    return result;
}

// Оптимизация памяти: O(n) времени, O(1) памяти
int fib_opt(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

Видите разницу? От O(2^n) до O(n) — это не оптимизация, это спасение.

## Задача 1: Рюкзак 0/1 (0/1 Knapsack)

Помните дробный рюкзак из лабораторной про жадные алгоритмы? Там можно было брать часть предмета, и жадный алгоритм работал идеально. Теперь предметы неделимы: либо берём целиком, либо не берём вообще. И тут жадность ломается.

### Постановка задачи

Дано N предметов, каждый с весом `weight[i]` и стоимостью `value[i]`. Рюкзак выдерживает вес W. Найти максимальную стоимость предметов, которые можно унести.

### Рекуррентное соотношение

```
dp[i][w] = максимальная стоимость, используя первые i предметов при ограничении веса w

dp[i][w] = max(
    dp[i-1][w],                           // не берём i-й предмет
    dp[i-1][w - weight[i]] + value[i]     // берём i-й предмет (если влезает)
)

База: dp[0][w] = 0 для всех w (нет предметов — нет стоимости)
```

### Требования к реализации

```c
typedef struct {
    int weight;
    int value;
    char name[64];
} Item;

typedef struct {
    Item* items;
    int size;
} ItemList;

typedef struct {
    int totalValue;
    int totalWeight;
    int* taken;      // taken[i] = 1, если i-й предмет взят
    int takenCount;
} KnapsackResult;

// Решение методом табуляции
// Возвращает максимальную стоимость и список взятых предметов
KnapsackResult* knapsack01(ItemList* items, int capacity);

// Вывод таблицы DP (для отладки и отчёта)
void printDPTable(int** dp, int n, int capacity);

// Восстановление решения по таблице
void reconstructSolution(int** dp, ItemList* items, int capacity, int* taken);
```

## Задача 2: Расстояние Левенштейна (Edit Distance)

Практическая задача, которую вы используете каждый день, даже не зная об этом. Когда телефон исправляет "привет" на "превет" — это расстояние Левенштейна. Когда Git показывает diff — там тоже оно. Когда поисковик понимает, что вы имели в виду — угадайте что.

### Постановка задачи

Даны две строки. Найти минимальное количество операций для превращения первой строки во вторую. Допустимые операции:
- **Вставка** символа
- **Удаление** символа
- **Замена** символа

### Рекуррентное соотношение

```
dp[i][j] = минимальное расстояние между s1[0..i-1] и s2[0..j-1]

Если s1[i-1] == s2[j-1]:
    dp[i][j] = dp[i-1][j-1]  // символы совпали, ничего не делаем
Иначе:
    dp[i][j] = 1 + min(
        dp[i-1][j],      // удаление из s1
        dp[i][j-1],      // вставка в s1
        dp[i-1][j-1]     // замена
    )

База:
    dp[i][0] = i  // удалить все символы из s1
    dp[0][j] = j  // вставить все символы из s2
```

### Требования к реализации

```c
typedef enum {
    OP_NONE,
    OP_INSERT,
    OP_DELETE,
    OP_REPLACE
} OperationType;

typedef struct {
    OperationType type;
    int position;
    char oldChar;
    char newChar;
} EditOperation;

typedef struct {
    int distance;
    EditOperation* operations;
    int operationCount;
} EditResult;

// Вычисление расстояния
int levenshteinDistance(const char* s1, const char* s2);

// Вычисление расстояния + восстановление операций
EditResult* levenshteinWithOperations(const char* s1, const char* s2);

// Вывод таблицы DP
void printEditTable(int** dp, const char* s1, const char* s2);

// Красивый вывод последовательности операций
void printOperations(EditResult* result, const char* s1, const char* s2);
```

## Задача 3 (hard): Наидлиннейшая общая подпоследовательность (LCS)

Подпоследовательность — это последовательность символов, полученная удалением некоторых (возможно, нуля) символов из строки без изменения порядка оставшихся. Например, "ACE" — подпоследовательность "ABCDE", а "AEC" — нет.

### Постановка задачи

Даны две строки. Найти длину их наидлиннейшей общей подпоследовательности (LCS) и саму подпоследовательность.

### Рекуррентное соотношение

```
dp[i][j] = длина LCS для s1[0..i-1] и s2[0..j-1]

Если s1[i-1] == s2[j-1]:
    dp[i][j] = dp[i-1][j-1] + 1  // символ входит в LCS
Иначе:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  // берём лучший вариант без одного из символов

База: dp[i][0] = dp[0][j] = 0
```

### Требования к реализации

```c
typedef struct {
    int length;
    char* subsequence;
    int** dpTable;        // для визуализации
    int rows, cols;
} LCSResult;

// Вычисление LCS
LCSResult* longestCommonSubsequence(const char* s1, const char* s2);

// Восстановление подпоследовательности
char* reconstructLCS(int** dp, const char* s1, const char* s2, int i, int j);

// Визуализация: подсветка LCS в обеих строках
void highlightLCS(const char* s1, const char* s2, const char* lcs);

void freeLCSResult(LCSResult* result);
```
