# Лабораторная работа №12 — Графы

---

### Цель работы

Реализовать поиск кратчайшего пути в графе на языке C, разобраться как работает Dijkstra и применить это на реальных данных улично-дорожной сети Санкт-Петербурга.

---

### Задание

Вам выдаются два файла с данными:

* `data/nodes.csv` — узлы графа: `id, lat, lon`
* `data/edges.csv` — рёбра: `from, to, length, oneway, name`

А также скрипт `find_ways.py` — он принимает два адреса, геокодирует их и запускает ваш бинарник.

#### 1. Спроектировать структуры данных

Никаких заготовок нет — это осознанно. Подумайте сами: как хранить граф? Что такое узел, ребро? Какие структуры из предыдущих лабораторных вам могут пригодиться?  
Отдельно уточню - обязательно переиспользовать ранее реализованные структуры из прошлых лабораторных работ. Если вам нужен список - используете список из прошлых лаб (однако разрешены любые модификации)  
**Если будет список специально для этой задачи - такое приниматься не будет!**

#### 2. Реализовать чтение графа из CSV

Парсите файлы сами. Узлы и рёбра нужно загрузить в память и организовать так, чтобы по `node_id` можно было быстро получить узел и его соседей.

#### 3. Реализовать алгоритм Дейкстры

Найдите кратчайший путь между двумя точками графа. Ближайшие к переданным координатам узлы — это ваша точка старта и финиша.

#### 4. Реализовать бинарник

Программа должна принимать аргументы командной строки:

```
./path_processor <path_to_data_dir> <path_to_input> <path_to_output>
```

* `path_to_data_dir` — папка с `nodes.csv` и `edges.csv`
* `path_to_input` — файл с двумя строками вида `lat lon`
* `path_to_output` — файл, куда записывается найденный путь (каждая строка: `lat lon`)

#### Формат выходного файла

Каждая строка — один узел пути, в порядке от старта до финиша:

```
<lat> <lon>
<lat> <lon>
...
```

Требования:
- разделитель — один пробел
- 6 знаков после запятой: `%.6lf`
- кодировка UTF-8, перевод строки Unix (`\n`)
- нет пробелов в конце строки
- все узлы пути включая первый и последний
- если путь не найден — файл пустой

Пример:
```
59.934206 30.335165
59.936115 30.336067
59.937359 30.336433
```

Тесты закрытые. Сравниваются SHA-256 хэши выходных файлов.

#### 5. Собрать отчёт

Запустите `find_ways.py` для **двух разных маршрутов** и приложите скриншоты карт с найденными путями. Без воды — только карты и выводы.

## Продвинутый уровень — A*

**Только для тех, кто выполнил базовое задание.**

Реализуйте алгоритм A* и найдите с его помощью **альтернативный маршрут** (отличный от основного). Что именно считать "альтернативным" — решаете сами, но аргументируйте выбор в отчёте.

На карте должны отображаться **три маршрута**: основной (Дейкстра) и два альтернативных (A*).
