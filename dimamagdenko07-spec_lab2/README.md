# lab11
Лабораторная работа №11 - Жадники

Итак, мы добрались до жадных алгоритмов. Нет, это не про то, как делить пиццу на вечеринке (хотя и там жадность — рабочая стратегия). Жадные алгоритмы — это подход, при котором на каждом шаге мы делаем локально оптимальный выбор в надежде получить глобально оптимальное решение.

Звучит как план "разберёмся по ходу дела", но удивительным образом для некоторых задач это работает идеально. А для некоторых — нет. И вот понять, когда жадность — это хорошо, а когда приведёт к катастрофе — это и есть цель данной лабораторной.

---

### Цель работы

Реализовать классические жадные алгоритмы, понять условия их применимости и на практике увидеть, когда жадный подход даёт оптимальное решение, а когда проваливается.

---

### Теоретическая справка

Жадный алгоритм строит решение пошагово, на каждом шаге выбирая лучший вариант **прямо сейчас**, не заглядывая в будущее. Это как шопинг без списка — берёшь то, что понравилось, и надеешься, что в итоге всё сложится.

**Когда жадный алгоритм работает:**
1. **Жадный выбор** — локально оптимальный выбор ведёт к глобально оптимальному решению
2. **Оптимальная подструктура** — оптимальное решение задачи содержит оптимальные решения подзадач

**Классические примеры:**
- Выбор активностей (Activity Selection) — всегда берём задачу, которая заканчивается раньше
- Алгоритм Хаффмана — всегда объединяем два символа с наименьшей частотой
- Дробный рюкзак — берём предметы с лучшим соотношением цена/вес
- Задача о сдаче монетами — берём монету максимального номинала (работает не для всех систем номиналов)

**Когда жадность подводит:**
- Целочисленный рюкзак (0/1 Knapsack) — жадный выбор не гарантирует оптимум
- Задача коммивояжёра — ближайший сосед часто даёт плохой маршрут

## Задача 1: Планирование мероприятий (Activity Selection)

Вы — организатор конференц-зала. К вам приходят заявки на бронирование: каждая заявка содержит время начала и время окончания. Два мероприятия не могут проходить одновременно. Ваша задача — принять максимальное количество заявок.

### Постановка задачи

Дан массив мероприятий, каждое с временем начала `start` и окончания `end`. Необходимо выбрать максимальное количество непересекающихся мероприятий.

### Жадная стратегия

Всегда выбираем мероприятие, которое **заканчивается раньше всех** (среди тех, что не конфликтуют с уже выбранными). Почему это работает? Чем раньше освободится зал, тем больше шансов вместить следующее мероприятие.

### Требования к реализации

```c
typedef struct {
    int start;
    int end;
    char name[64];
} Activity;

typedef struct {
    Activity* items;
    int size;
    int capacity;
} ActivityList;

// Создание и освобождение списка активностей
ActivityList* createActivityList(int capacity);
void freeActivityList(ActivityList* list);

// Добавление активности в список
void addActivity(ActivityList* list, int start, int end, const char* name);

// Основная функция — жадный выбор активностей
// Возвращает новый список с выбранными активностями
ActivityList* selectActivities(ActivityList* activities);

// Вспомогательная функция сортировки по времени окончания
void sortByEndTime(ActivityList* activities);
```

### Пример работы

```
Входные данные:
  "Лекция по алгоритмам"    09:00 - 10:30
  "Митинг команды"          09:30 - 11:00
  "Код-ревью"               10:30 - 11:30
  "Обед"                    11:00 - 12:00
  "Презентация проекта"     11:30 - 13:00
  "Ретроспектива"           12:00 - 12:30

Результат жадного алгоритма:
  1. "Лекция по алгоритмам"  09:00 - 10:30
  2. "Код-ревью"             10:30 - 11:30
  3. "Ретроспектива"         12:00 - 12:30

Выбрано 3 мероприятия (максимум для данного входа)
```

### Доказательство оптимальности (для отчёта)

В отчёте нужно объяснить, почему жадный выбор "берём то, что заканчивается раньше" всегда даёт оптимальное решение. Подсказка: используйте метод "обмена" — покажите, что любое оптимальное решение можно преобразовать в жадное без уменьшения количества активностей.

## Задача 2: Дробный рюкзак (Fractional Knapsack)

Вы — вор (гипотетически, конечно). Перед вами N предметов, каждый с весом и стоимостью. Ваш рюкзак выдерживает вес W. Но есть нюанс: предметы можно дробить! Хотите взять полкило золота — пожалуйста.

### Постановка задачи

Дан набор предметов с весом и стоимостью. Необходимо набрать максимальную стоимость, не превышая грузоподъёмность рюкзака. Предметы можно брать частично.

### Жадная стратегия

Сортируем предметы по **удельной стоимости** (цена / вес) в порядке убывания. Берём предметы целиком, пока влезают. Последний предмет берём частично, если целиком не помещается.

### Требования к реализации

```c
typedef struct {
    double weight;
    double value;
    char name[64];
} Item;

typedef struct {
    Item* items;
    int size;
    int capacity;
} ItemList;

// Результат: сколько какого предмета взяли
typedef struct {
    char name[64];
    double fraction;  // Доля предмета (от 0.0 до 1.0)
} TakenItem;

typedef struct {
    TakenItem* items;
    int size;
    double totalValue;
} KnapsackResult;

// Основная функция
KnapsackResult* fractionalKnapsack(ItemList* items, double capacity);

// Вспомогательные функции
double getValuePerWeight(Item* item);
void sortByValuePerWeight(ItemList* items);
```

### Пример работы

```
Грузоподъёмность рюкзака: 50 кг

Предметы:
  "Золото"    10 кг, 600$  (60 $/кг)
  "Серебро"   20 кг, 400$  (20 $/кг)
  "Бронза"    30 кг, 300$  (10 $/кг)

Жадный выбор (по убыванию $/кг):
  1. Золото:  10 кг целиком → 600$
  2. Серебро: 20 кг целиком → 400$
  3. Бронза:  20 кг из 30   → 200$ (взяли 66.7%)

Итого: 50 кг, 1200$
```

### Эксперимент: сравнение с целочисленным рюкзаком

Реализуйте также **наивное решение для целочисленного рюкзака** (полный перебор или рекурсия) и покажите на примере, что жадный алгоритм НЕ работает для 0/1 варианта:

```
Грузоподъёмность: 50 кг

Предметы:
  A: 10 кг, 60$  (6 $/кг)
  B: 20 кг, 100$ (5 $/кг)
  C: 30 кг, 120$ (4 $/кг)

Жадный (дробный): A + B + 2/3*C = 60 + 100 + 80 = 240$ ✓ (оптимально для дробного)
Жадный (0/1):     A + B = 160$ (рюкзак 30 кг)
Оптимальный (0/1): B + C = 220$ (рюкзак 50 кг) ← жадный проиграл!
```

## Задача 3 (hard): Кодирование Хаффмана

А теперь серьёзная задача с реальным применением. Алгоритм Хаффмана используется в JPEG, MP3, ZIP и куче других форматов. Суть: часто встречающиеся символы кодируем короткими битовыми последовательностями, редкие — длинными.

### Постановка задачи

Дан текст. Необходимо построить оптимальный префиксный код (код Хаффмана) и закодировать текст. Префиксный код означает, что никакой код не является префиксом другого — это позволяет однозначно декодировать.

### Жадная стратегия

1. Подсчитать частоту каждого символа
2. Создать лист для каждого символа (частота = приоритет)
3. Пока в очереди больше одного узла:
   - Извлечь два узла с минимальной частотой
   - Создать новый узел с частотой = сумма частот детей
   - Добавить новый узел обратно в очередь
4. Оставшийся узел — корень дерева Хаффмана
5. Коды символов определяются путём от корня: влево = 0, вправо = 1

### Требования к реализации

```c
typedef struct HuffmanNode {
    char symbol;           // '\0' для внутренних узлов
    int frequency;
    struct HuffmanNode* left;
    struct HuffmanNode* right;
} HuffmanNode;

typedef struct {
    char symbol;
    char code[256];        // Битовая строка как текст: "0110"
    int codeLength;
} HuffmanCode;

typedef struct {
    HuffmanCode* codes;
    int size;
    HuffmanNode* root;     // Корень дерева для декодирования
} HuffmanTable;

// Подсчёт частот символов в тексте
int* countFrequencies(const char* text, int* uniqueCount);

// Построение дерева Хаффмана
HuffmanNode* buildHuffmanTree(const char* text);

// Генерация таблицы кодов из дерева
HuffmanTable* generateCodes(HuffmanNode* root);

// Кодирование текста
char* encode(const char* text, HuffmanTable* table);

// Декодирование (для проверки)
char* decode(const char* encoded, HuffmanNode* root);

// Освобождение памяти
void freeHuffmanTree(HuffmanNode* root);
void freeHuffmanTable(HuffmanTable* table);
```

### Реализация приоритетной очереди
