# Задача 1: Планирование мероприятий (Activity Selection)

## Описание

Дан набор мероприятий с временем начала и окончания. Выбрать максимальное количество непересекающихся мероприятий.

## Жадная стратегия

**Всегда выбираем мероприятие, которое заканчивается раньше всех** (среди тех, что не конфликтуют с уже выбранными).

Почему это работает? Чем раньше освободится ресурс, тем больше шансов вместить следующее мероприятие.

## API

```c
typedef struct {
    int start;
    int end;
    char name[64];
} Activity;

typedef struct {
    Activity *items;
    int size;
    int capacity;
} ActivityList;

// Создание и добавление
ActivityList *createActivityList(int capacity);
int addActivity(ActivityList *list, int start, int end, const char *name);

// Сортировка по времени окончания
void sortByEndTime(ActivityList *list);

// Жадный выбор — возвращает новый список
ActivityList *selectActivities(ActivityList *activities);

// Визуализация
void printTimeline(ActivityList *all, ActivityList *selected);
```

## Сложность

- **Время:** O(n log n) — сортировка
- **Память:** O(n) — копия списка

## Доказательство оптимальности

Методом обмена: пусть есть оптимальное решение, где первой выбрана не та активность, что заканчивается раньше всех. Можно заменить её на ту, что заканчивается раньше, и получить решение не хуже (следующие активности по-прежнему не конфликтуют). Повторяя, получаем жадное решение.
