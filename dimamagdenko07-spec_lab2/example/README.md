# Введение в жадные алгоритмы

Жадный алгоритм — это когда вы на каждом шаге выбираете то, что кажется лучшим **прямо сейчас**, не думая о последствиях. Звучит как плохой жизненный совет, но для некоторых задач это идеальная стратегия.

Главный вопрос: когда жадность работает, а когда ведёт к катастрофе?
 
---

## Задача: Размен монет

Классический пример для понимания жадных алгоритмов.

**Условие:** У вас есть монеты номиналом 1, 5, 10, 25 копеек (как в США — penny, nickel, dime, quarter). Нужно выдать сдачу N копеек минимальным количеством монет.

### Жадная стратегия

Всегда берём монету **максимального номинала**, которая не превышает оставшуюся сумму.

```
Сдача: 41 копейка

Шаг 1: 41 >= 25? Да. Берём 25. Осталось: 16
Шаг 2: 16 >= 25? Нет. 16 >= 10? Да. Берём 10. Осталось: 6
Шаг 3: 6 >= 10? Нет. 6 >= 5? Да. Берём 5. Осталось: 1
Шаг 4: 1 >= 5? Нет. 1 >= 1? Да. Берём 1. Осталось: 0

Результат: 25 + 10 + 5 + 1 = 4 монеты
```

### Реализация

```c
// coins_greedy.c
#include <stdio.h>

int coinChangeGreedy(int amount, int *coins, int num_coins, int *result)
{
    int count = 0;

    // Предполагаем, что coins отсортированы по убыванию
    for (int i = 0; i < num_coins && amount > 0; i++)
    {
        while (amount >= coins[i])
        {
            result[count++] = coins[i];
            amount -= coins[i];
        }
    }

    return (amount == 0) ? count : -1;  // -1 если невозможно
}

int main()
{
    int coins[] = {25, 10, 5, 1};  // Отсортированы по убыванию
    int num_coins = 4;

    int amount = 41;
    int result[100];

    int count = coinChangeGreedy(amount, coins, num_coins, result);

    printf("Сдача %d копеек:\n", amount);
    printf("Монеты (%d шт.): ", count);
    for (int i = 0; i < count; i++)
    {
        printf("%d ", result[i]);
    }
    printf("\n");

    return 0;
}
```

Вывод:
```
Сдача 41 копеек:
Монеты (4 шт.): 25 10 5 1
```

## Когда жадность работает?

Для стандартных денежных систем (1, 5, 10, 25 или 1, 2, 5, 10) жадный алгоритм **всегда даёт оптимальное решение**. Это не случайность — такие системы специально спроектированы, чтобы жадность работала.

Почему? Потому что каждый следующий номинал "покрывает" предыдущие:
- 5 = 5 × 1 (никогда не выгодно брать 5 единиц вместо одной пятёрки)
- 10 = 2 × 5 (никогда не выгодно брать две пятёрки вместо десятки)
- и т.д.

---

## Когда жадность НЕ работает?

Возьмём другую систему монет: **{1, 3, 4}**

```
Сдача: 6 копеек

Жадный алгоритм:
  6 >= 4? Да. Берём 4. Осталось: 2
  2 >= 4? Нет. 2 >= 3? Нет. 2 >= 1? Да. Берём 1. Осталось: 1
  1 >= 1? Да. Берём 1. Осталось: 0

Жадный результат: 4 + 1 + 1 = 3 монеты

Оптимальный результат: 3 + 3 = 2 монеты ← ЛУЧШЕ!
```

Жадный алгоритм проиграл, потому что "заглянуть вперёд" он не умеет. Он не знает, что взяв 4, он лишит себя возможности использовать две тройки.

### Демонстрация

```c
// coins_fail.c
#include <stdio.h>

int coinChangeGreedy(int amount, int *coins, int n, int *result)
{
    int count = 0;
    for (int i = 0; i < n && amount > 0; i++)
    {
        while (amount >= coins[i])
        {
            result[count++] = coins[i];
            amount -= coins[i];
        }
    }
    return (amount == 0) ? count : -1;
}

// Оптимальное решение через DP
int coinChangeDP(int amount, int *coins, int n)
{
    int *dp = calloc(amount + 1, sizeof(int));
    for (int i = 1; i <= amount; i++)
        dp[i] = amount + 1;  // "бесконечность"

    dp[0] = 0;

    for (int i = 1; i <= amount; i++)
    {
        for (int j = 0; j < n; j++)
        {
            if (coins[j] <= i && dp[i - coins[j]] + 1 < dp[i])
            {
                dp[i] = dp[i - coins[j]] + 1;
            }
        }
    }

    int result = (dp[amount] > amount) ? -1 : dp[amount];
    free(dp);
    return result;
}

int main()
{
    int coins[] = {4, 3, 1};  // Проблемная система
    int n = 3;
    int amount = 6;

    int greedy_result[100];
    int greedy_count = coinChangeGreedy(amount, coins, n, greedy_result);

    int optimal_count = coinChangeDP(amount, coins, n);

    printf("Система монет: {4, 3, 1}\n");
    printf("Сумма: %d\n\n", amount);

    printf("Жадный алгоритм: %d монет (", greedy_count);
    for (int i = 0; i < greedy_count; i++)
        printf("%d%s", greedy_result[i], i < greedy_count - 1 ? " + " : "");
    printf(")\n");

    printf("Оптимальный (DP): %d монет (3 + 3)\n", optimal_count);

    if (greedy_count > optimal_count)
        printf("\n⚠️  Жадный алгоритм дал неоптимальный результат!\n");

    return 0;
}
```

## Свойства задач, где жадность работает

Жадный алгоритм гарантированно даёт оптимум, если задача обладает двумя свойствами:

### 1. Жадный выбор (Greedy Choice Property)

Локально оптимальный выбор ведёт к глобально оптимальному решению. То есть, выбрав лучший вариант сейчас, мы не закрываем путь к лучшему решению в будущем.

### 2. Оптимальная подструктура (Optimal Substructure)

Оптимальное решение задачи содержит оптимальные решения подзадач.

---

## Сравнение подходов

| Подход | Когда использовать | Сложность |
|--------|-------------------|-----------|
| **Жадный** | Есть доказательство оптимальности | Обычно O(n log n) |
| **Динамическое программирование** | Перекрывающиеся подзадачи | O(n × W) или O(n²) |
| **Полный перебор** | Маленький вход, нет лучшего способа | O(2^n) или O(n!) |

---

## Классические задачи, где жадность работает

1. **Activity Selection** — выбор максимального числа непересекающихся интервалов
2. **Дробный рюкзак** — можно брать часть предмета
3. **Кодирование Хаффмана** — оптимальное префиксное кодирование
4. **Минимальное остовное дерево** (Прим, Краскал)
5. **Кратчайший путь** (Дейкстра для неотрицательных весов)

---

## Классические задачи, где жадность НЕ работает

1. **0/1 Рюкзак** — предметы неделимы
2. **Задача коммивояжёра** — ближайший сосед даёт плохой маршрут
3. **Размен монет** (для некоторых систем номиналов)
4. **Longest Path** в общем графе

---

## Алгоритм применения жадного подхода

1. **Определите жадную стратегию** — что значит "лучший выбор" на каждом шаге?
2. **Докажите корректность** (или найдите контрпример)
   - Метод обмена: покажите, что любое оптимальное решение можно преобразовать в жадное
   - Или: покажите, что жадный выбор не закрывает путь к оптимуму
3. **Реализуйте алгоритм**
4. **Проверьте на контрпримерах**

## Переход к задачам лабораторной

В этой лабораторной вы реализуете три классических жадных алгоритма:

- **Activity Selection** — берём мероприятие, которое заканчивается раньше
- **Дробный рюкзак** — берём по убыванию ценность/вес
- **Кодирование Хаффмана** — объединяем два узла с минимальной частотой

Для каждого нужно понять, почему жадная стратегия даёт оптимум.

---

**P.S.** Жадность — это не порок, а O(n log n) вместо O(2^n). Но прежде чем применять жадный алгоритм, убедитесь, что он действительно даёт оптимум. Иначе получите решение, которое "почти хорошее", но на собеседовании этого не оценят.
