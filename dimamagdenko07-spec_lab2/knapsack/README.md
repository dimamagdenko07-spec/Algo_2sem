# Задача 2: Дробный рюкзак (Fractional Knapsack)

## Описание

Дано N предметов с весом и ценностью. Рюкзак выдерживает вес W. **Предметы можно брать частично.** Найти максимальную ценность.

## Жадная стратегия

Сортируем по **удельной стоимости** (ценность / вес) в порядке убывания. Берём предметы целиком, пока влезают. Последний — частично.

## API

```c
typedef struct {
    double weight;
    double value;
    char name[64];
} Item;

typedef struct {
    char name[64];
    double fraction;    // Доля (0.0 — 1.0)
    double weight;
    double value;
} TakenItem;

typedef struct {
    TakenItem *items;
    int size;
    double totalValue;
    double totalWeight;
} KnapsackResult;

// Жадный алгоритм
KnapsackResult *fractionalKnapsack(ItemList *items, double capacity);

// Для сравнения: 0/1 рюкзак (наивный)
double knapsack01Naive(ItemList *items, double capacity);
```

## Пример

```
Предметы:
  A: 10кг, 60$ (6 $/кг)
  B: 20кг, 100$ (5 $/кг)
  C: 30кг, 120$ (4 $/кг)

Вместимость: 50 кг

Результат:
  A: 100% (10кг) → 60$
  B: 100% (20кг) → 100$
  C: 66.7% (20кг) → 80$

ИТОГО: 50кг, 240$
```

## Сравнение с 0/1 рюкзаком

| | Дробный | 0/1 |
|---|---------|-----|
| Жадный | **240** (оптимум) | 160 (A+B, неоптимально!) |
| DP | — | **220** (B+C, оптимум) |

**Вывод:** Жадный работает только для дробного рюкзака!

## Сложность

- **Время:** O(n log n) — сортировка
- **Память:** O(n)

## Почему жадный работает для дробного?

Если мы можем брать часть предмета, то всегда выгодно брать как можно больше того, что имеет лучшее соотношение ценность/вес. Никогда не возникает ситуации, когда "взяв сейчас лучшее, мы теряем возможность взять ещё лучше потом".
